## 📄 研究论文

- **Language Models Need Sleep** 📅 2026-05-26 | arXiv:2605.26099
  作者/机构：Sangyun Lee 等
  核心贡献：(1) 提出"睡眠式巩固"机制，模型周期性地将近期上下文转化为持久快速权重并清空 KV 缓存；(2) 探索了状态空间模型（SSM）块中通过学习局部规则更新快速权重；(3) 增加睡眠轮次 N 可显著提升深度推理任务的表现。
  摘要：Transformer 大语言模型的注意力机制随上下文长度增长计算开销急剧上升。本文提出一种类似"睡眠"的巩固机制：模型周期性地将近期上下文转为持久快速权重，然后清空 KV 缓存。在睡眠阶段，模型对累积上下文执行 N 次离线循环处理并通过学习到的局部规则更新 SSM 块的快速权重。推理时，额外计算仅在睡眠时发生，保持推理延迟不变。在细胞自动机、多跳图检索和数学推理等任务上测试，常规 Transformer 及 SSM-注意力混合模型均失败，而增加睡眠持续时间可提升模型表现，在需要更深层推理的样本上收益最大。
  链接：[arXiv](https://arxiv.org/abs/2605.26099)

- **Looped Diffusion Language Models (LoopMDM)** 📅 2026-05-26 | arXiv:2605.26106
  作者/机构：Sanghyun Lee 等
  核心贡献：(1) 发现选择性循环 Transformer 早中层可显著提升掩码扩散语言模型的训练效率和性能；(2) 推理时可通过调整循环次数实现灵活的计算缩放；(3) GSM8K 上最高提升 8.5 分。
  摘要：掩码扩散模型（MDM）作为自回归模型的替代方案正在兴起，但其 Transformer 架构设计尚欠探索。本文提出 LoopMDM，通过选择性循环 Transformer 早中层实现两大收益：训练时循环产生深度缩放效果但不增加参数量，推理时改变循环次数可灵活缩放计算量。LoopMDM 以少至 3.3 倍训练 FLOPs 达到同尺寸 MDM 的性能，最终在 GSM8K 等推理基准上领先最多 8.5 分。即使在可比单步计算量下，LoopMDM 也超越更深层的非循环 MDM。注意力分析表明，循环在 MDM 中有效的原因是促进被遮蔽位置之间的交互。
  链接：[arXiv](https://arxiv.org/abs/2605.26106)

- **Claw-Anything：全方位评测 Always-On 个人助手** 📅 2026-05-26 | arXiv:2605.26086
  作者/机构：Yusong Lin 等（HKUST/蚂蚁集团等）
  核心贡献：(1) 提出沿三个维度扩展 Agent 上下文的基准：长跨度活动历史、互依赖后端服务、多设备 GUI+CLI 交互；(2) 通过多轮事件注入模拟数月用户活动；(3) GPT-5.5 仅获 34.5% pass@1。
  摘要：大语言模型 Agent 被越来越多地设想为全天候个人助手，可访问用户数字世界的所有相关信息。然而当前系统仅覆盖数字世界很窄的切片，现有基准同样只提供部分用户状态。本文提出 Claw-Anything 基准，沿三个维度扩展 Agent 上下文：长跨度活动历史、互依赖后端服务、跨多设备的 GUI 和 CLI 交互。通过多轮事件注入模拟数月用户活动，产生复杂世界状态和现实噪声（包括无关事件和冲突信号）。Agent 必须在丰富上下文环境中推理并保持对噪声的鲁棒性。实验显示 GPT-5.5 仅达 34.5% pass@1，远低于此前基准，揭示了当前 Agent 能力与全天候助手需求之间的巨大差距。
  链接：[arXiv](https://arxiv.org/abs/2605.26086)

- **LLMs 能编码因果方向，但其 Yes/No 输出无法表达（Causal Tongue-Tie）** 📅 2026-05-26 | arXiv:2605.25891
  作者/机构：Ziyi Ding 等
  核心贡献：(1) 发现线性探针可从模型隐状态恢复正确因果答案（准确率约 97%），但口头 Yes/No 回答却返回常识答案（约 50%）；(2) 定义了"Causal Tongue-Tie"现象，约 +0.5 的差距；(3) 对纯输出因果基准的结论提出质疑。
  摘要：本文发现大语言模型关于因果问题的编码与输出之间存在不一致。在反常识 CLadder 测试项上，固定线性探针可从模型隐状态恢复有证据支持的答案（准确率约 0.97），而口头 Yes/No 则回归常识答案（准确率约 0.5）。我们将这一约 +0.5 的差距称为"Causal Tongue-Tie"：错误的 Yes/No 可分解为两个可分离的失败模式——无内部信号 vs 有信号但语言接口无法表达。这对纯输出因果基准的含义是双向的：基准"正确"不一定意味着模型理解了，基准"错误"不一定意味着模型不能理解。仅从单一准确率数字对 LLM 因果推理能力做出断言需要重新审视。
  链接：[arXiv](https://arxiv.org/abs/2605.25891)

- **When Gradients Collide：多目标提示词优化为 LLM 评审官的失败模式** 📅 2026-05-26 | arXiv:2605.26046
  作者/机构：Abhishek Divekar 等（ACL 2026 Workshop）
  核心贡献：(1) 测试了五种文本梯度优化器的分解模式；(2) 发现 6/10 配置中优化从未超过初始提示词；(3) 识别了两个可分离失败模式：梯度稀释与指令干扰。
  摘要：定制 LLM 评审官通常涉及同时优化多个评估标准的提示词。文本梯度方法可自动化单评判标准的优化，但产生自然语言批评而非数值向量，因此多任务学习的冲突解决工具（PCGrad、MGDA）无法应用于多目标文本梯度场景。本文测试五种文本梯度优化器分解模式，发现在 6/10 配置中优化从未改善初始提示词。当梯度 LLM 联合处理多标准时，梯度特异性下降 59%（从 9.0 至 3.7）。将各任务指令简单合并为单个提示词使 Spearman's rho 下降 5.3%。识别出两个失败模式：优化时梯度稀释和推理时指令干扰，共同约束了多目标评审定制的设计空间。
