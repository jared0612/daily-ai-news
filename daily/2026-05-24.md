## 📄 研究论文

- **Gated DeltaNet-2: Decoupling Erase and Write in Linear Attention** 📅 2026-05-22 | arXiv:2605.22791
  作者/机构：Ali Hatamizadeh 等 / NVIDIA Labs
  核心贡献：(1) 提出通道级独立的擦除门和写入门，解耦了线性注意力中旧内容擦除与新内容写入的标量耦合；(2) 推导了含通道衰减的非对称擦除因子的分块 WY 算法，支持高效并行训练；(3) 在 1.3B/100B token 规模上全面超越 Mamba-2、Gated DeltaNet、KDA 和 Mamba-3 变体。
  摘要：线性注意力用固定大小的循环状态替代 softmax 注意力的无限缓存，将序列混合降到线性时间、解码降到常数内存。Delta-rule 模型在写入新值前减去当前读取，KDA 通过通道衰减锐化遗忘，但两者都用单一标量门同时控制擦除和写入。Gated DeltaNet-2 引入通道级擦除门 b_t 和写入门 w_t，是 Gated DeltaNet 和 KDA 的统一推广：当两门坍缩为同一标量时退化为 KDA，进一步坍缩衰减时退化为 Gated DeltaNet。论文推导了快速权重更新视角、含通道衰减的分块 WY 算法以及保持高效训练的门感知反向传播。在 1.3B 参数、100B FineWeb-Edu token 训练下，Gated DeltaNet-2 在语言建模、常识推理和检索任务上取得综合最优，在 RULER 长上下文大海捞针基准上优势尤为显著。
  链接：[arXiv](https://arxiv.org/abs/2605.22791) | [GitHub](https://github.com/NVlabs/GatedDeltaNet-2)

- **Advancing Mathematics Research with AI-Driven Formal Proof Search** 📅 2026-05-22 | arXiv:2605.22763
  作者/机构：George Tsoukalas, Anton Kovsharov, Pushmeet Kohli, Swarat Chaudhuri 等 / Google DeepMind
  核心贡献：(1) 首次大规模评估 LLM 生成 Lean 形式化证明解决开放数学问题的能力；(2) 最强智能体自主解决 353 个开放 Erdős 问题中的 9 个、44/492 个 OEIS 猜想；(3) 基础智能体（LLM 生成+Lean 验证交替）能复现 Erdős 成功但成本更高，揭示智能体设计对效率的关键影响。
  摘要：大语言模型在数学推理上日益出色，但其不可靠性限制了在数学研究中的效用。一种缓解方式是让 LLM 在 Lean 等形式化语言中生成形式化证明。本文对这种方法解决开放问题的能力进行了首次大规模评估。最强智能体以每个问题数百美元的成本自主解决了 9/353 个开放 Erdős 问题和 44/492 个 OEIS 猜想，正在被部署于组合数学、优化、图论、代数几何和量子光学研究中。基础智能体通过交替 LLM 生成与 Lean 验证可以复现 Erdős 问题的成功，但在最难题目上成本更高。这些发现展示了 AI 辅助形式化证明搜索的威力，并揭示了使其成为可能的智能体设计要素。
  链接：[arXiv](https://arxiv.org/abs/2605.22763)

- **Self-Evolution through Source-Level Rewriting in Autonomous Agent Systems (MOSS)** 📅 2026-05-22 | arXiv:2605.22794
  作者/机构：Qianshu Cai 等
  核心贡献：(1) 论证源码级适配是比文本可修改工件更通用的自我进化介质，因为它图灵完备、确定性生效且不受长上下文漂移侵蚀；(2) 提出 MOSS 系统，在生产级 Agent 基板上执行源码级自我重写，通过自动收集生产故障证据驱动的多阶段管线完成进化；(3) 在 OpenClaw 上将四任务平均评分从 0.25 提升至 0.61，无需人工干预。
  摘要：自主 Agent 系统在部署后基本是静态的，不会从用户交互中学习，反复出现的故障要等到下次人工更新才能修复。现有的自我进化 Agent 将进化局限于文本可修改工件（技能文件、提示配置、记忆模式、工作流图），而让 Agent 线束代码保持不变。由于路由、钩子排序、状态不变量和调度逻辑存在于代码而非文本工件中，一整类结构性故障在文本层根本无法触及。本文认为源码级适配是更通用的进化介质：图灵完备、是所有文本可修改范围的严格超集、确定性生效且不被长上下文漂移侵蚀。MOSS 系统通过自动收集的生产故障批次锚定每次进化，经过确定性多阶段管线，将代码修改委托给可插拔的外部编码 Agent CLI，MOSS 保留阶段排序和裁决。候选版本通过在临时试用 Worker 中对批次重放来验证，然后通过用户同意门控的就地容器交换（含健康探测门控回滚）来提升。
  链接：[arXiv](https://arxiv.org/abs/2605.22794) | [GitHub](https://github.com/dav-joy-thon/MOSS)
