- **MLEvolve: 自进化的机器学习算法发现多智能体框架** 📅 2026-06-04 | arXiv:2606.06473
  作者/机构：Shangheng Du 等
  核心贡献：提出 Progressive MCGS（蒙特卡洛图搜索）+ Retrospective Memory 机制，解决 MLE Agent 的跨分支信息隔离和无记忆搜索问题；解耦策略规划与代码生成，支持长周期自我进化。
  摘要：MLEvolve 是一个基于 LLM 的自进化多智能体框架，用于端到端机器学习算法发现。通过将树搜索扩展为 Progressive MCGS，MLEvolve 实现跨分支信息流，并使用熵启发的渐进式调度从广域探索转向聚焦利用。Retrospective Memory 将冷启动领域知识库与动态全局记忆结合，支持任务特定经验检索和重用。在 MLE-Bench 上，MLEvolve 以 12 小时预算（标准运行时一半）达到 SOTA 平均奖牌率和有效提交率，在数学算法优化任务上甚至超越 AlphaEvolve，展现了强跨域泛化能力。

- **Goedel-Architect: 基于蓝图生成与精炼的形式化定理证明** 📅 2026-06-04 | arXiv:2606.06468
  作者/机构：Jui-Hui Chung, Sanjeev Arora 等（Princeton）
  核心贡献：提出依赖图蓝图（Blueprint）范式，先构建定义和引理的依赖图，再并行证明各节点；使用开源 DeepSeek-V4-Flash 达到 99.2% MiniF2F 和 75.6% PutnamBench pass@1。
  摘要：Goedel-Architect 是一个面向 Lean 4 的 Agent 框架，核心创新在于蓝图生成与精炼策略——先生成由定义和引理组成的依赖图蓝图，再用工具化证明器并行闭合各开放引理节点，失败引理驱动蓝图精炼。这与主流递归引理分解方法形成对比，避免在死胡同策略上低效循环。使用开源 DeepSeek-V4-Flash（284B-A13B）作为骨干，以低于可比流水线 500 倍的成本，在 MiniF2F-test 达 100%，PutnamBench 达 88.8%（597/672），并解决 IMO 2025 4/6、Putnam 2025 11/12、USAMO 2026 3/6。
  链接：[arXiv](https://arxiv.org/abs/2606.06468)

- **Vortex: 高效可编程稀疏注意力服务系统** 📅 2026-06-04 | arXiv:2606.06453
  作者/机构：Zhuoming Chen 等
  核心贡献：提出 Python 嵌入式前端语言 + 页中心张量抽象的稀疏注意力系统，AI Agent 可自动生成和优化稀疏注意力算法，在 GLM-4.7-Flash 上达 4.7 倍吞吐提升。
  摘要：稀疏注意力对于大语言模型长文本生成场景日益重要，但大规模部署和评估新算法仍高度工程密集。Vortex 结合了 Python 嵌入式前端语言和页中心张量抽象，可表达广泛的稀疏注意力算法，后端紧密集成现代 LLM 服务栈。AI Agent 使用 Vortex 自动生成和优化算法，最佳结果达 3.46 倍全注意力吞吐提升且保持精度。Vortex 还将稀疏注意力扩展到新兴架构，在 MLA 架构 GLM-4.7-Flash 上达 4.7 倍，在 229B 参数 MiniMax-M2.7 上达 1.37 倍（NVIDIA B200 GPU）。
  链接：[arXiv](https://arxiv.org/abs/2606.06453)
