## 📄 研究论文

- **Knowledge Index of Noah's Ark (KINA)** 📅 2026-06-04 | arXiv:2606.05104
  作者/机构：Sheng Jin, Ge Zhang 等（261 个细分学科的专家覆盖）
  核心贡献：引入 KINA，一个包含 899 道题目、覆盖 261 个细分学科的 LLM 知识基准测试。将学科代表性建模为覆盖率优化目标，证明贪心近似算法可达 (1-1/e) 近似比；同时证明锦标赛式奖金激励在评审质量上弱优于固定报酬。评估 13 家实验室的 42 个模型，头名 Gemini-3.1-Pro-Preview 仅达 53.17%，Claude-Opus-4.6 为 49.92%，GPT-5.4 为 48.55%，显示巨大提升空间。
  摘要：现有 LLM 知识基准测试面临三个问题：以规模驱动的设计未真正覆盖学科代表性、固定报酬标注允许偷懒共识、有限测试预算下排名不稳定。KINA 通过专家锚点覆盖优化和锦标赛激励机制解决这些问题，揭示了模型表现呈现分层结构而非平滑全序——小规模前沿梯队（>48%）、密集强模型带（38-45%）、低表现模型仅略高于 10% 随机基线。工具增强可带来最多 5.17 分提升，且报告 bootstrap 统计以避免过度解读相邻排名。
  链接：[arXiv](https://arxiv.org/abs/2606.05104)

- **AutoLab: 前沿模型能否解决长期自动化研发与工程任务？** 📅 2026-06-04 | arXiv:2606.05080
  作者/机构：Zichen Chen, Alex Pentland, Mengdi Wang 等（MIT/普林斯顿等多机构合作）
  核心贡献：提出 AutoLab 基准测试，包含 36 个专家策划的真实任务，横跨系统优化、谜题挑战、模型开发和 CUDA 内核优化四大领域。每个任务从正确但次优的基线出发，在严格时间预算内挑战 Agent 持续改进。评估 17 个前沿模型发现成功的决定性因素不是初始尝试的质量，而是持续基准测试、编辑和利用经验反馈的迭代能力。
  摘要：科学和工程进步本质上是长期迭代的：提出变更、运行实验、测量结果、持续优化。然而现有前沿模型基准主要评估单轮回复或短程 Agent 轨迹，无法捕捉长时间持续迭代的挑战。AutoLab 填补了这一空白，作为超长期闭环优化基准，Claude-Opus-4.6 展现了强大的长期优化能力，但大多数前沿模型要么过早终止，要么耗尽预算仅取得微弱进展。结果凸显了时间感知和持续迭代在自主 Agent 中的重要性。代码和评估工具已开源。
  链接：[arXiv](https://arxiv.org/abs/2606.05080) | [GitHub](https://github.com/autolabhq/autolab) | [官网](https://autolab.moe/)

- **StreamMA: 流式通信在多 Agent 推理中的革命性应用** 📅 2026-06-04 | arXiv:2606.05158
  作者/机构：Zhen Yang 等（多 Agent 推理系统研究）
  核心贡献：提出 StreamMA，一种流式多 Agent 推理系统，将每个推理步骤在生成后立即流式传输给下游 Agent，实现流水线并行从而降低端到端延迟。令人意外的是，这种流水线操作还提升了效果：因为多步推理质量不均匀，早期步骤更可靠，StreamMA 让下游 Agent 利用这些可靠步骤而非完整链路，避免了后期不可靠步骤的误导。
  摘要：传统多 Agent 推理系统采用"生成后传输"范式，端到端延迟随管线深度线性增长。StreamMA 通过流式传输打破这一瓶颈，在 8 个推理基准（数学、科学、代码）、两个前沿 LLM（Claude Opus 4.6 和 GPT-5.4）和三种拓扑结构（链、树、图）上全面超越基线，平均提升 7.3 个百分点，最高提升 22.4 个百分点。研究还发现"步骤级缩放定律"——增加每个 Agent 的步骤数持续同时提升效果和效率，这是一个全新的缩放维度。
  链接：[arXiv](https://arxiv.org/abs/2606.05158) | [项目页](https://zhenyangcs.github.io/StreamMA-website/)
