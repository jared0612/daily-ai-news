## 📄 研究论文

- **MOSS: Self-Evolution through Source-Level Rewriting in Autonomous Agent Systems** 📅 2026-05-22 | arXiv:2605.22794
  作者/机构：Qianshu Cai 等 / OpenClaw
  核心贡献：提出源码级自我改写范式，突破现有自进化 Agent 仅修改文本工件（skill files、prompts）的局限；实现 MOSS 系统，通过生产失败证据自动驱动源码修改、确定性验证管线和用户同意门控的容器热替换。
  摘要：现有自主 Agent 系统部署后本质上是静态的，自进化 Agent 的适应范围被限制在文本可变工件上，无法触及代码层面的路由、钩子排序、状态不变量等结构性故障。本文论证源码级适应是更通用的进化媒介——它是图灵完备的，是所有文本可变作用域的超集，确定性生效且不受长上下文漂移影响。MOSS 系统在 OpenClaw 上将四任务平均评分从 0.25 提升至 0.61，单周期无人工干预。代码已开源。
  链接：[arXiv](https://arxiv.org/abs/2605.22794) | [GitHub](https://github.com/dav-joy-thon/MOSS)

- **Gated DeltaNet-2: Decoupling Erase and Write in Linear Attention** 📅 2026-05-22 | arXiv:2605.22791
  作者/机构：Ali Hatamizadeh 等 / NVIDIA Labs
  核心贡献：提出通道级擦除门与写入门解耦机制，克服线性注意力中标量门同时控制擦除和写入的根本局限；推导快速权重更新视角和块级 WY 算法。
  摘要：线性注意力用固定大小循环状态替代 softmax 的无界缓存，核心难题不是"遗忘什么"，而是"如何编辑压缩记忆而不打乱已有关联"。Delta 规则模型在写入前减去当前读取，KDA 用通道衰减锐化遗忘，但两者都用单一标量门同时控制擦除和写入。Gated DeltaNet-2 引入通道级擦除门 b_t 和写入门 w_t，在 KDA 时两门坍缩为同一标量，在 Gated DeltaNet 时衰减也坍缩。在 1.3B 参数/100B FineWeb-Edu tokens 训练下，Gated DeltaNet-2 在语言建模、常识推理和检索任务上全面超越 Mamba-2、Gated DeltaNet、KDA 和 Mamba-3 变体，在长上下文 RULER 基准上优势最为显著。代码已开源。
  链接：[arXiv](https://arxiv.org/abs/2605.22791) | [GitHub](https://github.com/NVlabs/GatedDeltaNet-2)

- **LCGuard: Latent Communication Guard for Safe KV Sharing in Multi-Agent Systems** 📅 2026-05-22 | arXiv:2605.22786
  作者/机构：Sadia Asif 等
  核心贡献：首次形式化多 Agent LLM 系统中 KV 缓存共享的信息泄露问题，提出 LCGuard 对抗训练框架，通过重建攻击定义安全性。
  摘要：LLM 多 Agent 系统越来越多地通过 KV 缓存进行隐式通信以提高效率，但 KV 缓存同时编码上下文输入、中间推理状态和 Agent 特定信息，形成敏感内容跨 Agent 无文本披露传播的不透明通道。LCGuard 将共享 KV 缓存视为隐式工作记忆，学习表示级变换，在传输前修改缓存内容。安全性通过对抗性重建定义：若对抗解码器可从共享缓存中恢复 Agent 特定敏感输入，则该缓存不安全。实验表明 LCGuard 在多种模型族和多 Agent 基准上一致降低重建泄露和攻击成功率，同时保持竞争性任务性能。
  链接：[arXiv](https://arxiv.org/abs/2605.22786)

- **HarnessAPI: A Skill-First Framework for Unified Streaming APIs and MCP Tools** 📅 2026-05-22 | arXiv:2605.22733
  作者/机构：Edwin Jose
  核心贡献：提出单一 typed skill 文件同时生成 HTTP 端点 + MCP 工具的 Python 框架，减少 74% 样板代码。
  摘要：每个部署为 LLM 工具的 Python 函数目前需要维护两套代码：HTTP 端点（人类/CI）和 MCP 工具注册（Agent 运行时），在路由、验证、序列化、流式传输等方面持续漂移。HarnessAPI 将 typed skill 文件夹作为单一真相来源，从一个 handler.py + Pydantic schema 自动派生 SSE 流式 HTTP 端点、OpenAPI/Swagger UI 和零配置 MCP 工具，所有服务运行在单一进程中。双模式内容协商让同一处理器无需修改即可服务 SSE 流式和 JSON 客户端。已在 PyPI 发布。
  链接：[arXiv](https://arxiv.org/abs/2605.22733) | [GitHub](https://github.com/edwinjosechittilappilly/harnessapi)
