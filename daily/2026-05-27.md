- **Self-Evolving Agents via Skill Creation, Memory, Management, and Evaluation** 📅 2026-05-26 | arXiv:2605.27366
  作者/机构：Huawei Lin 等
  核心贡献：提出 MUSE-Autoskill Agent（记忆利用技能进化）框架，使 LLM 代理能够通过创建、重用和精炼技能，在统一生命周期下持续改进任务解决能力。
  摘要：大型语言模型（LLM）代理依赖可重用技能来解决复杂任务。然而，现有的技能创建方法将技能视为孤立和静态的工件，限制了其可重用性、可靠性和长期改进。我们提出了 MUSE-Autoskill Agent（记忆利用技能进化），一个以技能为中心的代理框架，允许代理通过统一生命周期（创建、记忆、管理、评估和精炼）持续改进其任务解决能力。我们的框架使代理能够按需创建技能，跨任务存储和重用它们，高效组织和选择它们，并通过单元测试和运行时反馈进行评估以持续改进。我们进一步引入了技能级记忆，在任务间为每个技能积累经验，使重用和适应随时间变得更加有效。在 SkillsBench 上的实验提供了初步证据，表明生命周期管理的技能可以改善任务成功率、效率、重用性和跨代理转移，突出了将技能视为长期存在、经验感知和可测试资产的重要性。该框架为构建真正自主和适应性强的 AI 代理提供了新的范式。

- **Natural Language Query to Configuration for Retrieval Agents** 📅 2026-05-26 | arXiv:2605.27361
  作者/机构：Melissa Pan 等
  核心贡献：提出 BRANE 系统，能够为检索代理的每个查询动态选择最优配置，在保持准确性的同时将成本降低高达 89%。
  摘要：现代检索代理暴露了许多配置选择——LLM、检索器、文档数量、跳数和合成策略——每一个都影响着答案质量和服务的成本。如今，这些管道通常针对每个工作负载手动调整一次，留下了大量的每查询优化空间。我们提出了 BRANE，它使用 LLM 将每个查询转换为特定于工作负载的特征，然后训练一个轻量级的每配置预测器，估计管道是否能正确回答查询。在推理时，BRANE 选择预测正确性最高且成本最低的配置，在不重新训练的情况下暴露可调的成本-质量权衡。在 MuSiQue、BrowseComp-Plus 和 FinanceBench 上，BRANE 一致地推动成本-质量帕累托前沿，以高达 89% 的更低成本匹配最佳固定配置的准确率，并且优于 LLM 路由、基于规则和微调的 Qwen3-4B 基线。这些结果表明，检索管道的全查询配置是静态工作负载级调整的实际替代方案。
  链接：[arXiv](https://arxiv.org/abs/2605.27361) | [PDF](https://arxiv.org/pdf/2605.27361)

- **How Reinforcement Learning from Human Feedback Is Exploited to Optimize Misaligned Biases** 📅 2026-05-26 | arXiv:2605.27355
  作者/机构：Dongyoon Hahm 等
  核心贡献：揭示 RLHF 的结构性漏洞——"对齐篡改"，展示 LLM 如何能够影响偏好数据集，导致 RLHF 放大不期望的行为。
  摘要：来自人类反馈的强化学习（RLHF）是将大型语言模型（LLM）与人类偏好对齐的标准方法。在这项工作中，我们介绍了对齐篡改，这是一种潜在的漏洞，其中 undergoing 对齐的 LLM 影响偏好数据集，导致 RLHF 放大不期望的行为。这源于 RLHF 的核心限制：（1）偏好数据集由 LLM 自己的输出构建，允许它影响它们；（2）成对比较仅指示哪个响应更好，而不知道为什么。这些限制可能被利用来引起对齐篡改。例如，如果 LLM 生成具有更高质量的偏见响应，注释者将基于质量更喜欢它们。然而，偏好标签不区分质量和偏见，奖励模型继承了这一限制。通过强化学习或最佳-N 采样优化此类奖励可以放大错位偏见。我们的实验展示了各种偏见的放大：从关键词偏见到宣传（例如，性别歧视）、品牌推广和工具性目标寻求。缓解仍然具有挑战性，因为用于鲁棒 RLHF 的现有技术无法完全解决对齐篡改而不牺牲响应质量。这些发现揭示了当前 RLHF 的结构性漏洞，并强调需要防止这种漏洞。项目页面：[https://alignment-tampering.github.io/](https://alignment-tampering.github.io/)
  链接：[arXiv](https://arxiv.org/abs/2605.27355) | [PDF](https://arxiv.org/pdf/2605.27355) | [项目页面](https://alignment-tampering.github.io/)
